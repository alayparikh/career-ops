## Cross-cutting Advantage

Frame profile as:

**"Backend engineer with strong distributed systems and production experience"**

Key signals:
- Builds scalable microservices using Java and Spring Boot  
- Designs distributed systems using AWS (SNS, SQS, Lambda)  
- Strong in debugging, root-cause analysis, and reliability improvements  
- Delivers high-quality software using CI/CD pipelines  

---

## High-Conversion Positioning (CRITICAL)

Always prioritize these signals in matching and content generation:

1. **Scalability** → Handles high-throughput systems  
2. **Reliability** → Improves system stability and prevents failures  
3. **Distributed Systems** → Event-driven architecture (SNS/SQS/Lambda)  
4. **Production Experience** → Real-world systems, not academic projects  
5. **CI/CD & DevOps** → Automated pipelines and release quality  

---

## Resume Optimization Rules

When tailoring resume:

- Prefer **impact + system scale**
  - ✅ "Supported 1,000+ locations"
  - ❌ "Worked on backend APIs"

- Prefer **engineering signals**
  - debugging
  - system design
  - reliability
  - scalability

- Always include:
  - Java / Spring Boot
  - AWS (SNS, SQS, Lambda, RDS)
  - Microservices
  - CI/CD

---

## Cover Letter Strategy

Always include a cover letter if possible.

Structure:
1. Strong opening (role + fit)
2. 2–3 JD-aligned achievements
3. Tech stack match (Java, AWS, microservices)
4. Close with interest + impact

Keep under 1 page.

---

## Application Strategy (HIGH IMPACT)

Prioritize jobs with:
- Java + AWS + microservices
- Backend-heavy responsibilities
- Mid-level (2–5 years experience)

Avoid:
- Pure frontend roles
- Heavy C/C++ systems roles
- AI/ML-heavy roles without backend

- Prioritize **high-fit roles over volume**
